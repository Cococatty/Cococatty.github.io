Amazon is ON FIRE

This is an assignment I finished recently (try the full application at DATA423_Assignment2).

In this assignment, we were required to pick one out of four fields to conduct our own researches and present results appropriately.

Being inspired (heart-broken) by the Amazon fire incident, I wanted to talk about forest in this project. At the end, I have chosen Multi-level/multi-class classification and finished my project by using the data from Te Uru Rākau|Forestry New Zealand.

I have learnt HEAPS from this assignment, including but not limitted to:

Statistics

Different imputation methods IN PRACTICE

I learnt a lot about imputation methods and their implementation via this project!

Various modelling methods IN PRACTICE

I learnt a lot about imputation methods and their implementation via this project!

A more appropriate Data Science project design

I knew Data Science involves Analysis (Statistical analysis) and programming (Computer Science). But I did not pay much attention to the beginning and the end sections.

After completing this project, now I know at the beginning of a project, you MUST understand the project background, propose the core question, and then find relevant data. EDA is not a simple concept either. It turns out to be very important because it shows the hidden data characteristics (if right methods are applied!), it would guide you what to do next (missing data? class imbalance?). EDA is a skilled techniques because it requires attention to details, so that you can see the messages by using the right forms.

I had been underestimated the power of EDA, thought it is a generic process to ALL projects. But in this journey, I found EDA is similar to DNA - a pair might share exactly the same content, but the chance is pretty low (good for confidence interval estimation thou!).

I'm happy that I no longer using the same EDA techniques over my different projects, but start tailoring and thinking over it for EACH project!

Shiny

Dynamic number of plots x googleVis Package

I used googleVis package before but not in a very dynmaic manner. In this project, the googleVis plots are generated dynamically!

This means if the dataset has one extra factor variable, there will be one extra barchart generated AUTOMATICALLY!

More flexibilities in Shiny

Previously, I write report text in server.R, then use the output function in ui.R for display purpose. However, I found out I can write the text in ui.R with HTML tags! This is great because it has reduced unreuqired code and efficient has improved, at the same time, it has further extended my HTML skill!

Keep in mind of what you are doing!

During the development, my modelling methods were 'not' working when I tested in in Shiny application, thought they were working fine on local (with R session restarted).

I almost gave up on these modellings though it took me a while to write. But then I noticed I used the wrong outcome variable to these methods! After correcting the formulas to use the right outcome variables, everything RAN!

Next time I will write down the question on a piece of paper to remind myself what I am trying to solve so that I would not be lost because of heavy and intensive development.
